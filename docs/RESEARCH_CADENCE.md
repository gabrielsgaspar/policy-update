# Research Cadence

## Each Session

1. Read `RESEARCH.md`, root `TASKS.md`, and the topical docs relevant to the task.
2. Choose one mode: build, critique, execute, write, or review.
3. Execute at most one bounded research task unless the PI asks otherwise.
4. Validate the output.
5. Update `TASKS.md` and durable memory files as needed.

## Weekly

Run a research retrospective.

Questions:

- What did we learn?
- What did we try?
- What worked?
- What failed?
- What is the current strongest paper version?
- What is the biggest unresolved threat?
- What should we stop doing?
- What is next week's plan?

## Monthly

Run a viability review.

Questions:

- Is this still a paper?
- What is the publication path?
- What is the fatal unresolved issue?
- What would make us pivot, pause, or kill the project?
- Is the project still respecting privacy and raw-data immutability?
